Autonomous TVC Singlecopter

A drone with one rotor and no differential thrust. Every degree of freedom is recovered by redirecting the rotor's wake with four aerodynamic vanes — the same principle that steers rockets, built on a student budget.

Type
End-Year Engineering Project · INSAT (team of 4)
Timeline
2025 – 2026
Stack
Python · ESP32-S3 · KiCad · Blender · MQTT · React + Three.js
Validation
18/18 automated tests · SITL at 50 Hz

Why it's hard

A quadcopter steers by spinning four rotors at different speeds — an abundance of control authority. A singlecopter has exactly one fixed rotor, so it can climb and descend but not translate or turn. We restored full 6-DoF flight with thrust vector control: four flat vanes in the rotor's high-speed downwash deflect part of the wake, reacting steering forces back onto the airframe. Symmetric vane deflection translates the drone; differential deflection recovers yaw against the rotor's reactive torque.

From first principles to flight code

  • Aerodynamic modeling — momentum-theory thrust law, ground-effect correction, vane lateral-force model and dual-regime drag, all derived analytically and implemented one-to-one in the simulator and flight controller.
  • Cascaded control architecture — outer proportional position loops feeding inner PID velocity loops, analytically inverted through the vane model into deflection angles and throttle, with reactive-torque feed-forward for drift-free heading hold.
  • Hardware design — full propulsion sizing (thrust-to-weight, disc loading, motor KV selection), mechanical vane-actuation kinematics, and a custom PCB designed in KiCad around an ESP32-S3 with a 400 ms wireless fail-safe.
  • Digital twin — a physics-accurate Blender simulation speaking the same MQTT message contract as the real hardware, so the identical autopilot code flies both. Hundreds of autonomous missions were completed virtually before firmware ever touched the ESP32.
  • Live telemetry dashboard — a React + Three.js web app mirroring the drone in 3D at 50 Hz with per-term PID profiling charts.

Results

  • Autonomous point-to-point flight: a 17.2 m mission with a 3 m climb reaches its arrival radius in ~5.2 s and converges to < 1 cm final position error (software-in-the-loop).
  • Altitude hold settles within ±2% and < 2 cm steady-state error.
  • Entire control stack pinned down by an automated pytest suite — 18/18 passing — covering the PID law, force directions, autonomous climb/descent and the MQTT wire format.
